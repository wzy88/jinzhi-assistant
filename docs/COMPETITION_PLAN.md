# 津智助理比赛执行计划

## 1. 当前阶段判断

当前日期是 2026 年 5 月 20 日，比赛第一阶段已经进入初赛冲刺期。项目策略不应再按“赛前慢准备”推进，而应按“5-7 天完成可提交材料 + 尽快拿试点意向”推进。

## 2. 初赛提交目标

初赛需要让评委相信三件事：

1. 这是一个真实基层治理痛点。
2. 已经有可运行 Demo，而不是 PPT 概念。
3. 后续可以拿到真实试点数据。

如果按 OPC 赛道讲述，初赛还需要让评委相信第四件事：

4. 这不是单点网页，而是一个超级个体从立项、需求、设计、开发、测试、交付、运营到商业化的完整项目闭环。

## 3. 5 天冲刺计划

### Day 1

- 完成 MVP 页面。
- 完成 PRD、技术设计、部署说明。
- 明确项目一句话和演示脚本。
- 补齐 OPC 项目包：立项、需求、排期、设计、交互、测试、交付、变现、复盘。

### Day 2

- 部署到 Vercel。
- 绑定 `jinzhi.22dhmv.top`。
- 接入 DashScope Key 或确认使用规则引擎兜底。

### Day 3

- 整理 10-30 条脱敏/模拟工单样例。
- 制作 1 分钟演示视频。
- 记录分类结果和人工判断结果。
- 使用 `data/sample_tickets.csv` 跑一轮准确率评测。

### Day 4

- 联系社区、物业、校内后勤或宿管中心。
- 争取访谈记录、试用意向、反馈截图。
- 补充场景痛点描述。

### Day 5

- 完成初赛方案文档。
- 完成项目介绍页或 PDF。
- 准备报名材料。
- 把 OPC 项目包压缩成 3-5 页路演材料，突出“单人 + AI 全链路交付”。

## 4. 社区联系话术

您好，我正在参加天津市首届 AIGC 应用创新大赛，项目方向是“社区网格 AI 助理”。  

我们想免费为社区试用一个小工具：居民诉求输入后，系统自动判断类别、紧急程度、情绪标签，并生成处理建议和居民通知。  

目前不需要接入真实系统，也不会收集个人隐私。只希望能请您提供几条脱敏后的常见诉求样例，或者安排一次 15 分钟访谈，帮助我们确认这个工具是否符合网格员日常工作需要。  

如果后续您觉得有价值，我们可以免费开放 3 个月试用，并在比赛材料中以“试用意向/访谈反馈”的方式呈现，不涉及商业收费。

## 5. 可替代试点对象

如果短期内联系不到社区，可优先联系：

- 物业客服
- 校内后勤
- 宿舍报修中心
- 园区物业
- 社团或学院办公室

这些场景都可以归入“工单预处理”和“基层服务效率提升”。

## 6. 复赛准备

复赛是 48 小时揭榜挂帅，优先选择与现有产品接近的微需求：

- 工单分类工具
- 社区通知模板生成器
- 活动通知生成器
- 巡查报告生成器
- 高频问题统计看板

提前准备可复用模块：

- 工单分析 prompt
- 通知生成 prompt
- 周报生成 prompt
- CSV 样例数据
- 演示视频模板

## 7. 决赛证据链

决赛前需要尽量补齐：

- 试点对象名称或匿名说明
- 脱敏工单样例数量
- 工具处理条数
- 人工复核准确率
- 平均处理时间对比
- 网格员/物业人员反馈
- 试用截图或访谈记录
- 后续采购或继续试用意向

## 8. 路演主线

建议路演顺序：

1. 基层网格员每天面对大量非标准诉求。
2. 人工分类、回复、整理材料耗时且容易遗漏风险。
3. 津智助理用 AI 完成预处理。
4. 现场演示一条消防通道投诉。
5. 展示通知生成和看板。
6. 展示试点反馈和效率对比。
7. 说明复制到社区、物业、园区的商业路径。

OPC 版本路演应增加第 8 点：

8. 展示一个人如何用 AI 完成产品经理、设计师、工程师、测试、交付和销售顾问的工作流。
