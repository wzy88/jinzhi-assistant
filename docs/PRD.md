# 津智助理 PRD

## 1. 项目定位

津智助理是一款面向社区网格员的 AI 工单预处理与通知生成网页工具。项目服务于 2026“津门智创・超级个体”天津市首届 AIGC 应用创新大赛，重点验证“单人 + AI”如何在基层治理场景中快速交付可用工具，并通过真实试点数据形成评审证据链。

## 2. 一句话描述

把居民诉求自动转化为可分类、可分派、可回复、可复盘的社区工单。

## 2.1 OPC 参赛定位

津智助理不只提交一个网页原型，还要提交一个完整的 OPC 项目闭环：从场景立项、用户研究、需求定义、设计交互、开发测试、社区交付、商业模式到工作流复盘，证明“单人 + AI”可以完成一个微型 SaaS 解决方案从 0 到 1 的全过程。

OPC 项目材料见 [docs/opc/README.md](./opc/README.md)。

## 3. 目标用户

### 主要用户

- 社区网格员
- 社区综合服务窗口工作人员
- 物业客服与物业工程人员

### 评审/体验用户

- 大赛评委
- 社区负责人
- 高校创新创业导师
- 潜在政府采购或试点合作方

## 4. 用户痛点

1. 居民诉求来源杂，文字表达不标准，人工判断类别耗时。
2. 工单紧急程度依赖经验，容易漏掉消防、安全、矛盾纠纷等高风险问题。
3. 网格员需要频繁写通知、回复、周报，重复劳动多。
4. 社区难以快速沉淀高频问题和效率数据，决策复盘依赖手工统计。

## 5. MVP 范围

### 5.1 工单智能分析

输入居民诉求文本，输出：

- 诉求类别
- 紧急程度：高 / 中 / 低
- 情绪标签
- 建议处理部门
- 工单摘要
- 建议处理步骤
- 置信度

### 5.2 通知一键生成

输入通知主题，输出：

- 正式版通知
- 居民群大白话版
- 短信版

### 5.3 试点数据看板

展示：

- 累计处理工单数
- 平均分析时间
- 分类准确率
- 高频问题 TOP5
- 决赛证据链待补材料

### 5.4 处理记录与人工复核

每次分析后自动生成处理记录，支持：

- 回看原始诉求与分析结果
- 标记处理状态：待转派 / 已转派 / 已回访
- 标记人工复核：待复核 / 正确 / 需调整
- 基于复核结果计算演示准确率

### 5.5 批量导入

支持导入 `.csv` 或 `.txt` 文件，将每行文本作为一条居民诉求生成演示工单。初赛阶段用于快速展示试点数据沉淀能力。

### 5.6 周报生成

基于当前工单记录生成社区工单周报草稿，包含总处理量、高优先级数量、复核准确率、高频问题和建议动作。

### 5.7 账号与个人中心

初赛阶段提供演示级账号体系，支持：

- 演示登录 / 退出登录
- 查看当前身份与认证状态
- 编辑姓名、手机号、所属机构、岗位角色、服务片区
- 模拟身份认证通过 / 取消认证
- 设置模型模式与隐私偏好

该版本使用浏览器本地存储保存资料和设置，不接入真实短信登录或服务端用户库。

## 6. 暂不做范围

以下功能不进入初赛 MVP：

- 多租户权限系统
- 复杂报表导出
- 实时聊天
- 完整工单流转审批
- 天津方言语音识别
- 自动拨打电话或短信发送
- 服务端持久化数据库
- 真实短信验证码登录
- 真实实名认证接口

这些功能可以作为复赛或决赛阶段扩展。

## 7. 核心使用流程

1. 网格员打开网页。
2. 粘贴居民诉求文本。
3. 点击“智能分析并入库”。
4. 系统输出类别、紧急程度、情绪、部门、摘要和处理步骤。
5. 系统自动沉淀处理记录。
6. 网格员对结果进行人工复核并更新处理状态。
7. 网格员基于分析结果生成居民通知并复制使用。
8. 负责人查看看板，了解高频问题和处理效率。
9. 生成周报草稿，用于复盘和汇报。

## 8. 比赛得分设计

### 政府代表 20%

得分点：

- 对应基层治理真实痛点。
- 可用于社区、物业、街道等实际场景。
- 支持脱敏数据和试点反馈。

### 行业专家 40%

得分点：

- AI 分类和生成能力可演示。
- 前端交互简洁，普通网格员无需培训即可使用。
- 架构简单，便于快速迭代和稳定部署。

### 投资人/企业家 40%

得分点：

- 可复制到更多社区、物业、园区。
- 可采用试点服务费、年度订阅、项目制交付等模式。
- 能展示“单人 + AI”快速完成产品、部署、材料和试点的完整工作流。

## 9. 成功指标

### 初赛指标

- 完成线上可访问 Demo。
- 完成 1 分钟演示视频。
- 获取 10 条以上脱敏样例或模拟样例。
- 获得至少 1 次社区/物业/校内后勤访谈记录。

### 复赛指标

- 可在 48 小时内适配工单分类、通知生成、周报生成类微需求。
- 分类准确率达到 85% 以上。
- 收集 1 份试用意向或反馈说明。

### 决赛指标

- 试点运行 1-2 周。
- 累计处理 50 条以上真实或脱敏工单。
- 人工复核准确率冲刺 90%。
- 形成效率对比报告和用户反馈截图。

## 10. 风险与应对

| 风险 | 应对 |
| --- | --- |
| 暂时拿不到社区试点 | 使用物业、校内后勤、宿舍报修等近似场景先验证 |
| 大模型结果不稳定 | 保留规则引擎兜底，限制输出 JSON 结构 |
| API Key 泄露 | Key 只放在服务端环境变量，不进入前端代码 |
| 评委质疑数据真实性 | 保存访谈记录、原始脱敏样例、处理前后截图 |
| 现场网络故障 | 准备本地规则引擎和录屏演示 |
