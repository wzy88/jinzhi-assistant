# 准确率评测模板

## 1. 评测目的

评测不是为了证明 AI 永远正确，而是为了证明津智助理在社区工单预处理场景中可用、可复核、可持续改进。

## 2. 评测数据

初赛阶段使用：

```text
data/sample_tickets.csv
```

当前 30 条为模拟脱敏样例。真实试点开始后，使用社区/物业提供的脱敏工单替换。

## 3. 评测表字段

| 字段 | 说明 |
| --- | --- |
| id | 样例编号 |
| text | 居民诉求 |
| human_category | 人工标注类别 |
| ai_category | AI 输出类别 |
| category_correct | 分类是否正确，1/0 |
| human_urgency | 人工标注紧急度 |
| ai_urgency | AI 输出紧急度 |
| urgency_correct | 紧急度是否正确，1/0 |
| notice_usable | 通知是否可用，1/0 |
| reviewer_note | 人工复核备注 |

## 4. 评测流程

1. 导入 `data/sample_tickets.csv`。
2. 对每条诉求执行“智能分析并入库”。
3. 记录 AI 输出类别和紧急度。
4. 与人工标注对比。
5. 标记分类是否正确、紧急度是否正确。
6. 对需要通知的样例生成通知，判断是否可直接使用或少量修改后使用。
7. 计算准确率。

## 5. 计算方式

```text
分类准确率 = category_correct 总和 / 样例总数
紧急度准确率 = urgency_correct 总和 / 样例总数
通知可用率 = notice_usable 总和 / 需要通知的样例数
综合可用率 = 可直接进入人工处理流程的样例数 / 样例总数
```

## 7. 评委表达口径

如果使用模拟数据：

> 初赛阶段我们用 30 条模拟脱敏样例验证了评测流程，下一步会用社区或物业提供的真实脱敏工单替换。项目从第一天就设计了人工复核和准确率统计，不把模型结果当成不可质疑的黑盒。

如果已有真实脱敏数据：

> 我们使用某社区/物业提供的脱敏工单进行回测，人工标注作为基准，统计分类准确率、紧急度准确率和通知可用率。所有公开材料都已去除个人隐私信息。

## 8. 错误分析模板

| 错误类型 | 例子 | 改进方式 |
| --- | --- | --- |
| 类别混淆 | 积水同时可能是环境卫生和设施维修 | 增加二级类别或主次类别 |
| 紧急度偏低 | 电梯故障被判为中 | 增加高风险关键词 |
| 通知语气不合适 | 居民群版太正式 | 增加本地表达样例 |
| 部门建议不准 | 纠纷转给物业而非社区调解 | 增加部门规则表 |
