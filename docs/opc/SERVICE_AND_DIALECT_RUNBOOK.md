# 服务端与方言识别落地说明

## 当前线上结构

- 代码仓库：GitHub `wzy88/jinzhi-assistant`
- 前端部署：Vercel 自动构建 `dist`
- 服务端接口：Vercel Serverless Functions，代码位于 `api/`
- 当前接口：
  - `/api/analyze`：居民诉求分类、优先级、摘要、建议动作
  - `/api/notice`：正式版、微信群版、短信版通知生成
  - `/api/transcribe`：演示版方言转写文本规范化

## 为什么不是单独买一台服务器

比赛演示和早期试点阶段，Vercel 的 Serverless 接口足够承载：

- 评委和社区人员直接打开网页使用
- 前端与服务端一起跟随 GitHub 自动部署
- 不需要先维护云服务器、Nginx、证书和运维脚本

如果进入真实社区试点，可以再迁移到腾讯云或阿里云：

- 前端：继续 Vercel，或迁到腾讯云 COS + CDN
- 服务端：腾讯云 CloudBase / 云函数 / 轻量服务器
- 数据库：腾讯云数据库或 Supabase/Postgres
- 文件存储：腾讯云 COS，用于语音、图片、附件

## 方言识别怎么处理

当前 Demo 不直接上传音频，而是先模拟“语音已被转写成文本”的后半段流程：

1. 前端收集转写文本和口音类型
2. `/api/transcribe` 规范口语、方言词和表达
3. 规范后的文本进入 `/api/analyze`
4. 网格员复核、转派、生成通知

真实试点时替换第 1-2 步：

1. 居民或网格员上传语音
2. 调用 ASR 服务识别音频
3. 对天津话、普通话、混合口音做热词和纠错
4. 输出规范工单文本
5. 进入现有分析和处置流程

候选 ASR：

- 腾讯云语音识别：适合腾讯云部署和政企采购链路
- 阿里云智能语音交互：适合继续使用 DashScope / 通义体系
- 科大讯飞：中文语音和方言能力强，适合后续做专项评测

## 试点前需要补齐

- 10-30 条脱敏真实语音或转写样例
- 天津话常见词、楼栋名、小区名、物业名热词表
- ASR 准确率评测表：原文、识别结果、人工修正、错误类型
- 语音文件保存策略：是否保存原音频、保存多久、谁能访问
- 居民授权说明：录音用途、处理范围、删除机制
