# 商业模式

## 1. 商业定位

津智助理不是大型政务系统，而是面向社区、物业、园区和高校后勤的轻量 AI 工单预处理工具。商业模式应避免一开始就走重交付，优先用低成本试点换取反馈和转介绍。

## 2. 目标客户分层

| 客户 | 付费能力 | 购买理由 | 进入方式 |
| --- | --- | --- | --- |
| 单个社区 | 低-中 | 提升工作效率、形成治理数据 | 免费试用、街道推荐 |
| 物业项目 | 中 | 降低客服整理成本、提升响应速度 | 项目经理试用 |
| 物业公司 | 中-高 | 多项目复制、总部看板 | 从单项目扩展 |
| 高校/园区后勤 | 中 | 报修分类、通知生成、周报 | 后勤部门试点 |
| 街道/园区平台 | 高 | 多社区统一治理工具 | 试点报告后对接 |

## 3. 服务包设计

### 免费试点版

价格：0 元，1-3 个月。

包含：

- 网页 Demo 使用。
- 30-100 条脱敏数据评测。
- 1 次线上培训。
- 1 份试点反馈报告。

目的：换取真实反馈、截图、案例和推荐。

### 标准订阅版

参考价格：199-499 元/月/社区或物业项目。

包含：

- 工单分析。
- 通知生成。
- 数据保存。
- 周报生成。
- CSV 导入导出。
- 基础反馈表。

适合：单社区、单物业项目、学校后勤小团队。

### 项目交付版

参考价格：5000-20000 元/项目。

包含：

- 私有数据模板配置。
- 账号和权限。
- 试点数据看板。
- 试点报告导出。
- 1-2 次培训。
- 1 个月交付支持。

适合：街道、园区、物业公司试点。

### 定制扩展版

参考价格：按需求报价。

可能包含：

- 接入已有工单系统。
- 接入语音转文字。
- 方言识别。
- 多单位数据隔离。
- 私有化部署。

## 4. 获客路径

1. 从比赛资源获取社区或政务场景对接。
2. 用演示视频发给社区、物业、后勤联系人。
3. 以“免费试用，不接隐私数据，只用脱敏样例”为低门槛切入。
4. 用试点报告换取转介绍。
5. 从单点试用扩展到物业公司或街道多点试用。

## 5. 销售话术

```text
我们不是替代社区人员，而是帮网格员把居民诉求先整理成结构化工单。

您只需要提供几条脱敏后的常见诉求，我们可以免费帮您跑一次：
系统会自动给出类别、紧急程度、建议部门、处理步骤，还能生成居民通知和周报草稿。

如果您觉得有用，我们可以免费开放 3 个月试用。后续如果要正式使用，再讨论是否按月订阅或按项目交付。
```

## 6. 成本结构

| 成本 | 初赛阶段 | 正式使用 |
| --- | --- | --- |
| 域名 | 已有域名子域名 | 低 |
| 部署 | Vercel 免费/低成本 | 低 |
| 模型调用 | 少量 DashScope 调用 | 随使用量增加 |
| 数据库 | 初期无 | Supabase 免费/低成本起步 |
| 交付时间 | 单人投入 | 主要成本 |
| 客服支持 | 单人支持 | 可标准化 |

## 7. 收入闭环证明

比赛期间最低目标不是马上收钱，而是拿到以下任一证据：

- 继续试用意向。
- 推荐给其他社区/物业的意向。
- 愿意付费的口头反馈。
- 物业项目经理认可报价区间。
- 街道或园区愿意组织下一轮试点。

决赛前冲刺目标：

- 1 个免费试点。
- 1 份试点报告。
- 1 个付费意向或继续试用意向。

## 8. 竞争优势

| 维度 | 津智助理优势 |
| --- | --- |
| 轻量 | 无需接入复杂系统即可试用 |
| 聚焦 | 只做工单预处理、通知、周报 |
| 低成本 | 单人 + AI 交付，价格可控 |
| 可复制 | 社区、物业、后勤、园区都可复用 |
| 证据导向 | 从第一天就收集复核、效率和反馈数据 |

## 9. 商业风险

| 风险 | 应对 |
| --- | --- |
| 社区预算有限 | 先走免费试点和街道/物业公司转化 |
| 政务采购周期长 | 先服务物业和后勤等更快决策场景 |
| 数据隐私顾虑 | 只用脱敏数据，提供本地/私有化选项 |
| AI 准确率不稳定 | 保留人工复核和规则引擎 |
| 同质化工具多 | 用社区场景数据和交付 SOP 做差异 |
