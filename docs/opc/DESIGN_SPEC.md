# 设计规格

## 1. 设计目标

津智助理不是营销官网，而是社区网格员和物业人员可以直接使用的轻量工作台。设计目标是：

1. 首屏进入工作，不用学习复杂系统。
2. 结果结构化，便于人工确认。
3. 看板服务负责人复盘，不做装饰性数据。
4. 比赛评委能快速看出真实落地路径。

## 2. 设计原则

| 原则 | 说明 |
| --- | --- |
| 工作优先 | 把输入、分析、结果、复核放在核心区域 |
| 低培训成本 | 用明确按钮、可见 label、自然语言字段 |
| 人工可控 | AI 只做建议，复核与状态由人确认 |
| 证据导向 | 看板、周报、准确率都服务试点证明 |
| 演示可读 | 评委现场 60 秒内理解核心价值 |

## 5. 关键状态

| 状态 | 设计要求 |
| --- | --- |
| 空数据 | 显示样例和引导，不出现空白页 |
| 分析中 | 按钮 loading，避免重复点击 |
| API 失败 | 使用规则引擎兜底，不中断演示 |
| 复核正确 | 指标立即更新 |
| 需调整 | 记录为错误样例，用于后续优化 |
| 批量导入 | 导入后自动选中第一条 |
| 复制成功 | 按钮文字短暂显示已复制 |

## 6. 设计稿补充计划

初赛阶段可以使用产品截图 + 标注作为设计稿。复赛前建议补 Figma：

1. 首页工作台框架。
2. 工单分析结果状态。
3. 处理记录表格。
4. 试点看板。
5. 移动端单列布局。

## 7. 可访问性要求

- 所有表单必须有 label。
- 主要按钮高度不低于 44px。
- 结果区使用 `aria-live`。
- 色彩不能只依赖颜色表达状态。
- 移动端不应出现核心内容溢出。

## 8. 下一轮设计修正

1. 把设置里的“演示/真实数据”做成真正的数据模式。
2. 给准确率、节省时间、试用反馈增加专门证据区。
3. 增加“导出试点报告”入口。
4. 给社区第一次使用增加极简 3 步引导。
