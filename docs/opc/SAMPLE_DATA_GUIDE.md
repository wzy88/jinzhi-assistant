# 样例数据说明

## 1. 文件位置

样例数据位于：

```text
data/sample_tickets.csv
```

## 2. 字段说明

| 字段 | 说明 |
| --- | --- |
| id | 样例编号 |
| text | 脱敏或模拟后的居民诉求 |
| human_category | 人工标注类别 |
| human_urgency | 人工标注紧急程度 |
| source | 数据来源，当前为模拟 |
| notes | 标注说明 |

## 3. 使用方式

### 用于产品演示

1. 打开津智助理。
2. 点击“导入样例”。
3. 选择 `data/sample_tickets.csv`。
4. 查看处理记录和看板变化。
5. 对样例进行人工复核。

### 用于准确率评测

1. 导入样例。
2. 逐条查看 AI 输出。
3. 和 `human_category`、`human_urgency` 对比。
4. 标记正确或需调整。
5. 计算分类准确率和紧急度准确率。

## 4. 重要说明

当前 30 条数据是模拟样例，不应在路演中说成真实社区数据。可以这样表达：

> 初赛阶段我们先用 30 条模拟脱敏样例验证流程；下一步将用社区/物业提供的脱敏工单替换，并保留同样的人工复核评测方法。

## 5. 真实数据替换要求

真实社区或物业样例进入项目之前，必须处理：

- 删除居民姓名。
- 删除手机号。
- 删除身份证号。
- 模糊具体门牌号。
- 删除微信号、车牌号等可识别信息。
- 保留问题类型、处理场景和风险等级。

## 6. 评测目标

| 阶段 | 样例数量 | 分类准确率目标 | 紧急度准确率目标 |
| --- | --- | --- | --- |
| 初赛 | 30 条 | 80%+ | 80%+ |
| 复赛 | 50 条 | 85%+ | 85%+ |
| 决赛 | 100 条 | 90%+ | 90%+ |
