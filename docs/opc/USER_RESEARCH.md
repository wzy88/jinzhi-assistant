# 用户研究计划

## 1. 研究目标

通过轻量访谈和试用反馈，验证津智助理是否真正解决社区/物业/后勤人员的日常工单处理问题。

## 2. 目标访谈对象

| 优先级 | 对象 | 目的 |
| --- | --- | --- |
| P0 | 社区网格员 | 验证诉求分类、紧急度、通知生成是否贴近真实工作 |
| P0 | 物业客服 | 验证报修、投诉、分派、回访流程 |
| P1 | 社区负责人 | 验证看板、周报、试点数据是否有管理价值 |
| P1 | 校内后勤或宿管 | 作为社区不可得时的近似场景 |

## 3. 访谈问题

### 日常工作

1. 居民诉求主要从哪些渠道来？
2. 每天大约处理多少条诉求？
3. 最耗时间的是分类、分派、回复、回访还是统计？
4. 哪些类型诉求最容易漏判或升级？

### 对产品的反应

1. 看到 AI 输出的类别、紧急度、摘要和建议动作，哪些字段有用？
2. 哪些字段不可信或需要人工改？
3. 通知生成的正式版、居民群版、短信版是否符合实际使用？
4. 高频问题看板和周报能否用于汇报？

### 试点意向

1. 是否愿意用 10-30 条脱敏样例试一次？
2. 是否愿意让 1 位网格员试用 1 周？
3. 如果免费试用 3 个月，哪些条件必须满足？
4. 如果后续付费，社区/物业更能接受按月订阅、按项目交付还是一次性部署？

## 5. 试用反馈表

| 问题 | 选项 |
| --- | --- |
| 工单分类是否准确 | 准确 / 基本准确 / 不准确 |
| 紧急度判断是否合理 | 合理 / 偏高 / 偏低 |
| 摘要是否能直接使用 | 可以 / 需少量修改 / 不能用 |
| 建议动作是否有帮助 | 有帮助 / 一般 / 没帮助 |
| 通知文案是否能复制使用 | 可以 / 需少量修改 / 不能用 |
| 是否节省时间 | 明显节省 / 略有节省 / 没节省 |
| 是否愿意继续试用 | 愿意 / 看情况 / 不愿意 |

## 6. 需要沉淀的证据

1. 访谈对象数量和角色。
2. 脱敏工单原文数量。
3. AI 输出与人工判断对比。
4. 试用者反馈截图或文字确认。
5. 节省时间估算。
6. 是否有继续试用或采购意向。
