## 4. API 测试

### 工单分析

```bash
curl -X POST http://localhost:5173/api/analyze \
  -H "Content-Type: application/json" \
  -d '{"text":"12号楼消防通道被占用，救护车进不来。"}'
```

预期：

- HTTP 200
- 返回 JSON
- `urgency` 为 `高`
- `nextSteps` 不超过 3 条

### 通知生成

```bash
curl -X POST http://localhost:5173/api/notice \
  -H "Content-Type: application/json" \
  -d '{"prompt":"消防通道专项清理提醒","analysis":{"category":"消防安全"}}'
```

预期：

- HTTP 200
- 返回 `formal`、`friendly`、`sms`
- `sms` 尽量控制在 70 字内

## 5. 准确率评测

### 数据要求

- 至少 30 条脱敏样例。
- 每条包含人工类别和人工紧急度。
- AI 输出后由人工标记是否正确。

### 计算方式

```text
分类准确率 = 分类正确数量 / 已复核数量
紧急度准确率 = 紧急度正确数量 / 已复核数量
综合可用率 = 可直接使用或少量修改的数量 / 已复核数量
```

### 目标

| 阶段 | 分类准确率 | 紧急度准确率 | 综合可用率 |
| --- | --- | --- | --- |
| 初赛 | 80%+ | 80%+ | 75%+ |
| 复赛 | 85%+ | 85%+ | 80%+ |
| 决赛 | 90%+ | 90%+ | 85%+ |

## 6. UAT 测试

让社区/物业/后勤人员完成以下任务：

1. 输入一条真实脱敏诉求。
2. 判断 AI 分类和紧急度是否合理。
3. 生成一条通知并复制。
4. 标记处理状态和复核结果。
5. 查看周报草稿。

记录：

- 是否能独立完成。
- 完成耗时。
- 哪一步卡住。
- 哪个字段不可信。
- 是否愿意继续使用。
