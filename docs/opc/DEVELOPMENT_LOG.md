## 3. 核心技术决策

| 决策 | 原因 |
| --- | --- |
| React + Vite | 快速开发，便于部署和演示 |
| Vercel Serverless API | API Key 不暴露在前端 |
| DashScope 兼容 OpenAI 接口 | 贴合比赛支持单位和国内部署环境 |
| 规则引擎兜底 | 现场网络或模型失败时仍可演示 |
| 单页工作台 | 降低社区人员学习成本 |
| 本地状态优先 | 初赛快速验证，避免过早接复杂后端 |

## 4. 已知技术债

| 技术债 | 影响 | 优先级 |
| --- | --- | --- |
| 处理记录未服务端保存 | 真实试点数据不可跨设备追踪 | P0 |
| 设置项部分未真实生效 | 评委可能认为是假功能 | P0 |
| 自动脱敏仅基础规则 | 真实样例覆盖不足时仍有隐私风险 | P0 |
| 导出仅基础 TXT/CSV | 还缺可直接提交的 PDF/图表证据包 | P1 |
| 自动化测试覆盖较窄 | 已有 Playwright 冒烟测试，仍需覆盖更多异常路径 | P1 |
| 前后端规则引擎重复 | 维护成本增加 | P2 |

## 6. 交付检查

每次发布前执行：

```bash
npm run lint
npm run build
```

手动检查：

1. 首页打开。
2. 示例工单可分析。
3. 处理记录增加。
4. 人工复核更新准确率。
5. 通知生成正常。
6. 周报刷新正常。
7. 手机端可阅读。
