# OPC 项目立项书

## 1. 项目名称

津智助理：面向社区网格员的 AI 工单预处理与通知生成工作台。

## 2. 立项背景

2026“津门智创・超级个体”天津市首届 AIGC 应用创新大赛强调真实落地、商业闭环和单人 + AI 的交付能力。比赛的“智慧生态”赛道明确包含社区网格智能助手方向，要求解决政府治理、社区服务或中小企业的实际痛点。

社区网格员、物业客服和基层服务窗口每天面对大量非标准化居民诉求。这些诉求来自微信群、电话、线下登记、物业报修等渠道，表达方式不统一，人工分类、判断紧急度、写通知、做周报都需要消耗大量时间。

## 3. 机会判断

这个项目值得做，原因有三点：

1. 真实痛点明确：基层工单处理存在重复劳动和风险识别压力。
2. AIGC 适配度高：分类、摘要、通知生成、周报生成都是大模型擅长的文本结构化任务。
3. 单人交付可行：MVP 不需要接入复杂政务系统，先用脱敏数据和网页工作台即可验证。

## 4. 目标客户

| 客户类型 | 使用者 | 采购或决策者 | 价值 |
| --- | --- | --- | --- |
| 社区居委会 | 网格员、窗口人员 | 社区书记、街道负责人 | 提升诉求处理效率，降低高风险问题遗漏 |
| 物业公司 | 客服、工程调度 | 项目经理、物业公司负责人 | 提升报修分派效率，沉淀高频问题 |
| 高校/园区后勤 | 后勤客服、宿管 | 后勤部门负责人 | 降低报修整理和通知撰写成本 |

## 5. 核心假设

| 假设 | 验证方式 |
| --- | --- |
| 网格员愿意使用轻量网页工具处理诉求 | 让 1-3 位真实或近似用户试用 |
| AI 可以稳定完成常见诉求分类 | 用 30 条脱敏样例做人工复核评测 |
| 通知生成能节省重复写作时间 | 对比人工写通知与 AI 生成后的修改时间 |
| 社区负责人关心高频问题和周报 | 访谈负责人，确认看板字段是否有价值 |
| 物业/社区存在试点或采购可能 | 获取继续试用、推荐或付费意向 |

## 6. MVP 目标

在初赛阶段，MVP 只证明四件事：

1. 居民诉求可以快速结构化为类别、紧急度、情绪、建议部门、摘要和动作。
2. 网格员可以人工复核，形成准确率证据。
3. 系统可以生成多版本居民通知和周报草稿。
4. 项目可以通过公网链接给评委和社区人员直接体验。

## 7. 非目标

初赛阶段不做完整政务系统，不接真实个人隐私数据，不做复杂审批流，不自动发送短信，不替代社区人员最终判断。

## 8. 成功标准

| 阶段 | 成功标准 |
| --- | --- |
| 初赛 | 完成线上 Demo、演示视频、30 条样例、1 次访谈或试用意向 |
| 复赛 | 48 小时内适配具体微需求，分类准确率达到 85% 以上 |
| 决赛 | 试点运行 1-2 周，处理 50 条以上脱敏工单，复核准确率冲刺 90% |

## 9. OPC 价值证明

项目最终不是证明“我会写一个网站”，而是证明“我可以用 AI 承担一个小型解决方案公司的完整角色”：

- 产品经理：定义场景、需求和优先级。
- 设计师：完成工作台信息架构和交互。
- 工程师：开发前端、API 和部署。
- 测试人员：设计测试用例和准确率评测。
- 交付经理：制作社区使用 SOP 和试点材料。
- 销售顾问：设计试点转付费路径。
- 运营复盘者：形成效率对比和用户反馈报告。
